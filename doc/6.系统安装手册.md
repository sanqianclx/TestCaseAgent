# 系统安装手册

> **项目名称**：TestGenerate Agent
> **文档版本**：V1.0
> **更新日期**：2026-06-18
> **适用版本**：V3.1

---

## 1. 安装概述

本手册指导用户从零开始完成 TestGenerate Agent 的环境准备、依赖安装、配置与启动。系统提供两种使用方式：

- **CLI 模式**：命令行直接生成测试，适合脚本化与本地调试。
- **Web 模式**：浏览器图形界面，适合团队协作与历史管理。

两种模式共享同一套后端核心（Mastra Agent + 工作流），Web 模式额外需要 MySQL 数据库与前端服务。

### 1.1 安装流程总览

```
①环境准备 → ②获取源码 → ③安装 Node 依赖 → ④安装 Python 依赖
→ ⑤配置环境变量 → ⑥初始化数据库 → ⑦构建 → ⑧启动验证
```

## 3. 环境准备

### 3.1 安装 Node.js

1. 访问 [Node.js 官网](https://nodejs.org/) 下载 LTS 版本（>= 22）。
2. 运行安装程序，勾选"Add to PATH"。
3. 验证安装：

```bash
node --version   # 应输出 v22.x.x 或更高
npm --version
```

### 3.2 安装 Python

1. 访问 [Python 官网](https://www.python.org/downloads/) 下载 3.10+ 版本。
2. 安装时**勾选 "Add Python to PATH"**。
3. 验证安装：

```bash
python --version   # 应输出 Python 3.10+
pip --version
```

### 3.3 安装 MySQL（Web 模式）

1. 访问 [MySQL 官网](https://dev.mysql.com/downloads/) 下载 MySQL 8.0+ Community Server。
2. 安装时设置 root 密码（示例中为 `123456`，生产环境请用强密码）。
3. 验证安装：

```bash
mysql -u root -p
# 输入密码后进入 MySQL 提示符即成功
```

### 3.4 安装 Java 工具链（可选，仅测 Java）

1. 安装 [JDK 17+](https://adoptium.net/)。
2. 安装 [Maven](https://maven.apache.org/download.cgi)，配置 `MAVEN_HOME` 或 `M2_HOME` 环境变量。
3. 验证：

```bash
java -version
mvn -version
```

### 3.5 安装 C++ 工具链（可选，仅测 C++）

**Windows 推荐 MSYS2**：

1. 访问 [MSYS2 官网](https://www.msys2.org/) 安装 MSYS2。
2. 在 UCRT64 终端安装 g++ 与 GoogleTest：

```bash
pacman -S mingw-w64-ucrt-x86_64-gcc
pacman -S mingw-w64-ucrt-x86_64-gtest
```

3. 验证：

```bash
g++ --version   # 应支持 C++17
```

> 系统会自动解析 MSYS2 UCRT64 bin 目录并前置到 PATH，无需手动配置。也可通过设置 `GTEST_ROOT` 环境变量指定 GoogleTest 源码路径。

### 3.6 获取 DeepSeek API Key

1. 访问 [DeepSeek 开放平台](https://platform.deepseek.com/) 注册账号。
2. 创建 API Key，复制保存。
3. 系统使用 `deepseek-v4-flash`（快速生成）与 `deepseek-v4-pro`（高精度重试）两个模型通道。

## 4. 获取源码

```bash
git clone <仓库地址> testgenerate-agent
cd testgenerate-agent
```

确认目录结构包含 `src/`、`client/`、`prisma/`、`python-runtime/` 等核心目录。

---

## 5. 安装依赖

### 5.1 安装 Node 依赖（后端）

```bash
npm install
```

该命令安装 Mastra、Express、Prisma、DeepSeek SDK、Tree-sitter 等后端依赖。

### 5.2 安装 Python 依赖

```bash
pip install -r python-runtime/requirements.txt
pip install coverage
```

> `requirements.txt` 包含 `pytest>=8.0.0`；`coverage` 用于 Python 真实覆盖率测量，需单独安装。

### 5.3 安装前端依赖（Web 模式）

```bash
cd client
npm install
cd ..
```

该命令安装 React、Ant Design、Vite、Axios 等前端依赖。

## 6. 配置环境变量

### 6.1 创建配置文件

```bash
cp .env.example .env
```

### 6.2 编辑 .env 文件

打开 `.env`，按实际环境修改以下关键项：

```env
# 数据库配置（Web 模式必需，按你的 MySQL 实际修改）
DATABASE_URL="mysql://root:你的密码@localhost:3306/testgenerate"
DB_HOST=localhost
DB_PORT=3306
DB_USER=root
DB_PASSWORD=你的密码
DB_NAME=testgenerate

# JWT 配置（Web 模式必需，生产环境务必修改 secret）
JWT_SECRET="请替换为一串随机字符串"
JWT_EXPIRES_IN="2h"
JWT_REFRESH_EXPIRES_IN="7d"

# 服务器配置
PORT=3000
HOST=0.0.0.0
NODE_ENV=development

# CORS 配置（前端地址）
CORS_ORIGIN="http://localhost:5173"

# 文件上传配置
UPLOAD_DIR="./uploads"
MAX_FILE_SIZE=10485760

# LLM 配置（必需）
DEEPSEEK_API_KEY="你的 DeepSeek API Key"

# 日志配置
LOG_LEVEL=debug
LOG_DIR=./logs

# 安全配置
RATE_LIMIT_WINDOW_MS=900000
RATE_LIMIT_MAX=100
```

### 6.3 配置项说明

| 配置项 | 说明 | 必需 |
| :--- | :--- | :--- |
| DATABASE_URL | MySQL 连接串，Prisma 使用 | Web 模式 |
| DEEPSEEK_API_KEY | DeepSeek LLM 密钥 | ✅ |
| JWT_SECRET | JWT 签名密钥，生产必须修改 | Web 模式 |
| PORT | 后端服务端口，默认 3000 | Web 模式 |
| CORS_ORIGIN | 前端地址，默认 5173 | Web 模式 |
| TESTGENERATE_PYTHON_RUNTIME_DIR | Python 运行时目录（一般无需配置，系统自动定位） | 可选 |
| GTEST_ROOT | GoogleTest 源码路径（C++ 编译可选） | 可选 |

## 7. 初始化数据库（Web 模式）

### 7.1 创建数据库并建表

```bash
mysql -u root -p < docs/init-database.sql
```

该脚本创建 `testgenerate` 数据库及全部 10 张表（users、sessions、tasks、messages 等）。

### 7.2 生成 Prisma Client

```bash
npm run prisma:generate
```

> 若数据库结构有变更，可执行 `npm run prisma:migrate` 生成迁移，或 `npm run prisma:push` 直接同步。

### 7.3（可选）查看数据库

```bash
npm run prisma:studio
```

打开 Prisma Studio 可视化管理数据库内容。

## 9. 启动与验证

### 9.1 启动 CLI 模式

**方式一：交互模式（推荐新手）**

```bash
npm run generate -- --interactive
```

**方式二：命令行模式**

```bash
npm run generate -- --input ./src/example.py --output ./my-tests
```

**方式三：自主 Agent 模式**

```bash
npm run generate -- --autonomous
```

> 开发阶段也可用 `npm run agent` 直接进入自主模式。

### 9.2 启动 Web 模式

需要**两个终端**分别启动前后端。

**终端 1：启动后端**

```bash
npm run server:dev
```

后端运行在 `http://localhost:3000`，开发模式支持热重载。

**终端 2：启动前端**

```bash
cd client
npm run dev
```

前端运行在 `http://localhost:5173`。

### 9.3 启动 Mastra Studio（可选调试）

```bash
npx mastra dev
```

浏览器打开 `http://localhost:4111`，可在可视化面板调试 Agent 与 Workflow。

### 9.4 验证安装

| 验证项 | 方法 | 预期结果 |
| :--- | :--- | :--- |
| Node 环境 | `npm run build` 无报错 | dist/ 目录生成 |
| Python 环境 | `python -c "import pytest"` | 无报错 |
| LLM 连接 | 运行任意生成命令 | 不报 API Key 错误 |
| 数据库连接 | `npm run server:dev` 启动 | 控制台无数据库错误 |

## 10. 常见问题排查

### 10.1 Python 运行时找不到

**现象**：报错提示找不到 `python-runtime/` 目录或运行时脚本。

**解决**：
- 确认 `python-runtime/` 目录存在且含 `parse_source.py`、`run_pytest.py`、`export_cases.py`、`coverage_runner.py` 四个脚本。
- 或在 `.env` 中设置 `TESTGENERATE_PYTHON_RUNTIME_DIR` 指向该目录绝对路径。
- 确认 `python` 命令可用（Windows 可能需用 `py` 或配置 PATH）。

### 10.2 LLM 调用失败

**现象**：报 `assertLlmAvailable` 错误或 API Key 无效。

**解决**：
- 检查 `.env` 中 `DEEPSEEK_API_KEY` 是否正确。
- 确认 Key 不含非 ASCII 字符或多余空格。
- 确认 Key 非占位符（如 `your-deepseek-api-key`）。
- 检查网络能否访问 DeepSeek API。

### 10.3 数据库连接失败

**现象**：后端启动报 Prisma 连接错误。

**解决**：
- 确认 MySQL 服务已启动。
- 确认 `DATABASE_URL` 中的用户名、密码、端口正确。
- 确认已执行 `docs/init-database.sql` 建库建表。
- 确认执行过 `npm run prisma:generate`。

### 10.4 C++ 编译失败

**现象**：报找不到 `g++` 或 `gtest.h`。

**解决**：
- 确认 MSYS2 UCRT64 g++ 已安装且在 PATH。
- 或设置 `GTEST_ROOT` 指向 GoogleTest 源码目录。
- 系统会自动解析 UCRT64 bin 目录，若仍失败可手动将该目录加入系统 PATH。

### 10.5 Java 测试执行失败

**现象**：报 Maven 不可用或编译错误。

**解决**：
- 确认 `mvn` 命令可用，或配置 `MAVEN_HOME`/`M2_HOME`。
- 确认 JDK 17+ 已安装且 `JAVA_HOME` 配置正确。
- Maven 首次执行会下载依赖，需联网。

### 10.6 前端无法连接后端

**现象**：前端页面报跨域或接口 404。

**解决**：
- 确认后端已启动在 3000 端口。
- 确认 `.env` 中 `CORS_ORIGIN` 与前端地址一致（默认 `http://localhost:5173`）。
- 检查 `client/src/api/client.ts` 中的 baseURL 配置。

### 10.7 端口冲突

**现象**：启动报端口占用。

**解决**：修改 `.env` 中 `PORT`（后端）或 `client/vite.config.ts` 中的端口（前端）。
