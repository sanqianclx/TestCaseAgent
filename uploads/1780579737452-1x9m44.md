> 与之配套的 [TestSourceBuggyTen.java](./TestSourceBuggyTen.java) 中**故意不暴露**任何缺陷提示。

| 方法                 | 缺陷类型          | 严重度 | 触发难度 |
| ------------------ | ------------- | --- | ---- |
| `isValidEmail`     | 空指针 / 数组越界    | 中   | 易    |
| `parseAge`         | 空指针 / 类型转换    | 中   | 易    |
| `calculateTotal`   | 逻辑错误(负数)      | 中   | 中    |

| `isPalindrome`     | 空指针 / 大小写     | 中   | 中    |

## 详细缺陷说明

### 1. `isValidEmail(String email)`

| 缺陷 | 描述                                                                                 | 触发用例                   |
| -- | ---------------------------------------------------------------------------------- | ---------------------- |
| D1 | 未对 `email` 做 null 校验,直接 `email.contains("@")` 抛 NullPointerException               | `isValidEmail(null)`   |
| D2 | `email.split("@")[1]` 未校验数组长度,`"abc@"` 会使 `[1]` 越界抛 ArrayIndexOutOfBoundsException | `isValidEmail("abc@")` |

**修复建议**:先判 `email != null`;改用 `indexOf("@")` 与 `indexOf(".", atIndex)` 替代 `split`。

***

### 2. `parseAge(String input)`

| 缺陷 | 描述                                 | 触发用例                     |
| -- | ---------------------------------- | ------------------------ |
| D1 | 未处理 null 输入                        | `parseAge(null)`         |
| D2 | 非数字字符会抛 NumberFormatException,未做防御 | `parseAge("二十")`         |
| D3 | 未校验合法范围(年龄应 ≥ 0 且 ≤ 150 左右)        | `parseAge("-5")` 返回 `-5` |

**修复建议**:catch `NumberFormatException` 抛业务异常;校验范围。

***

### 3. `calculateTotal(double unitPrice, int quantity)`

| 缺陷 | 描述                         | 触发用例                                 |
| -- | -------------------------- | ------------------------------------ |
| D1 | 负数 `quantity` 仍参与计算,产生负数总价 | `calculateTotal(10.0, -3)` → `-30.0` |
| D2 | 负数 `unitPrice` 未拦截         | `calculateTotal(-5.0, 2)` → `-10.0`  |

**修复建议**:`if (unitPrice < 0 || quantity < 0) throw new IllegalArgumentException(...)`。

***

### 4. `getFullName(String firstName, String lastName)`

| 缺陷 | 描述                                           | 触发用例                     |
| -- | -------------------------------------------- | ------------------------ |
| D1 | 任一参数为 null 时,字符串拼接得到 `"null 张"` 或 `"李 null"` | `getFullName(null, "张")` |

**修复建议**:对 `firstName` / `lastName` 做 null/空字符串校验,或使用 `Objects.toString`。

***

### 5. `divide(int dividend, int divisor)`

| 缺陷 | 描述                           | 触发用例            |
| -- | ---------------------------- | --------------- |
| D1 | 除数为 0 时抛 ArithmeticException | `divide(10, 0)` |

**修复建议**:`if (divisor == 0) throw new IllegalArgumentException("divisor 不能为 0");`。

***

### 6. `findMax(int[] numbers)`

| 缺陷 | 描述                                                      | 触发用例                  |
| -- | ------------------------------------------------------- | --------------------- |
| D1 | `numbers` 为 null 时,`numbers[0]` 抛 NullPointerException  | `findMax(null)`       |
| D2 | 数组长度为 0 时,`numbers[0]` 抛 ArrayIndexOutOfBoundsException | `findMax(new int[0])` |
| D3 | 注释写"不可为 null",但实现未做防御,文档与实现不一致                          | —                     |

**修复建议**:`if (numbers == null || numbers.length == 0) throw new IllegalArgumentException(...)`。

***

### 7. `formatAmount(double amount)`

| 缺陷 | 描述                                         | 触发用例                       |
| -- | ------------------------------------------ | -------------------------- |
| D1 | 负数会输出 `-1,234.56`,业务上常需要 `(1,234.56)` 括号表示 | `formatAmount(-1234.5)`    |
| D2 | `NaN` 会输出 `NaN`,无防御                        | `formatAmount(Double.NaN)` |
| D3 | 隐式依赖 JVM 默认 Locale,不同地区千分位/小数点符号不同         | —                          |

**修复建议**:对负数加括号;对 `NaN`/`Infinity` 显式处理;显式传入 `Locale.US`。

***

### 8. `joinWithComma(String[] parts)`

| 缺陷 | 描述                                               | 触发用例                                                           |
| -- | ------------------------------------------------ | -------------------------------------------------------------- |
| D1 | 数组为 null 时,`parts.length` 抛 NullPointerException | `joinWithComma(null)`                                          |
| D2 | 元素含 null 时,拼接结果出现字面量 `"null"`                    | `joinWithComma(new String[]{"A", null, "C"})` → `"A, null, C"` |

**修复建议**:null 校验;使用 `String.join` 或 `Objects.toString(parts[i], "")`。

***

### 9. `isPalindrome(String text)`

| 缺陷 | 描述                                   | 触发用例                  |
| -- | ------------------------------------ | --------------------- |
| D1 | `text` 为 null 抛 NullPointerException | `isPalindrome(null)`  |
| D2 | 大小写未归一化,`"Aa"` 判为 false              | `isPalindrome("Aa")`  |
| D3 | 未去除首尾空白与内部空格,`"a a"` 判为 false        | `isPalindrome("a a")` |

**修复建议**:`text = text.toLowerCase().replaceAll("\\s+", "");`,并加 null 保护。

***

### 10. `countOccurrences(String text, String sub)`

| 缺陷 | 描述                                                                             | 触发用例                            |
| -- | ------------------------------------------------------------------------------ | ------------------------------- |
| D1 | `text` / `sub` 为 null 抛 NullPointerException                                   | `countOccurrences(null, "a")`   |
| D2 | **死循环**:`sub` 为空字符串时,`text.indexOf("", index)` 永远返回 `index`,循环不退出,最终 OOM / 栈溢出 | `countOccurrences("hello", "")` |
| D3 | `text` 为空字符串时返回 0,虽然正确但缺少显式保护                                                  | `countOccurrences("", "a")`     |

**修复建议**:`if (sub == null || sub.isEmpty()) return 0;`;对 null 做显式判空。

***

## 评分建议(可选)

- **必须抓到**:`divide` 除零、`countOccurrences` 死循环、`findMax` 空数组 —— 体现 Agent 对"经典坑"的敏感度
- **加分项**:`isValidEmail` 的 split 越界、`isPalindrome` 的大小写、`formatAmount` 的 NaN —— 体现 Agent 对"边角细节"的深度
- **完美通关**:全部 10 个方法的全部缺陷都被覆盖(包含正常/异常路径)
